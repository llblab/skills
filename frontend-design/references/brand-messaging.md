# Brand Messaging and Voice

Frontend design communicates through copy, hierarchy, labels, CTAs, visual proof, and interaction tone. Brand is not a separate layer; it is the reason the interface sounds and looks like itself. Coordinate message hierarchy with [`information-presentation.md`](information-presentation.md), visual expression with [`visual-identity-assets.md`](visual-identity-assets.md), and claim/provenance review with [`visual-craft.md`](visual-craft.md).

## Message Architecture

```text
Mission → Vision → Value Proposition → Positioning → Key Messages → Proof Points → CTAs
```

### Mission

`We [action] for [audience] by [method] so they can [outcome].`

### Value Proposition

`For [target customer] who [need/problem], [product] is a [category] that [key benefit]. Unlike [alternative], it [differentiator].`

### Positioning

`[Product] is the [category] for [audience] who want [outcome] because [reason to believe].`

## UI Messaging Rules

- One primary message per viewport or major section.
- Headlines say the change; subheads explain the mechanism.
- Buttons use verbs: Start, Create, Compare, Review, Export, Continue.
- Labels describe user intent, not implementation internals.
- Error copy states cause and recovery.
- Empty states explain what is missing and the next action.
- Tooltips clarify, never hide critical meaning.
- Proof should sit near claims: metrics, logos, screenshots, testimonials, compliance, case studies.

## Voice Dimensions

| Dimension | Range | UI Effect |
|---|---|---|
| Formality | Formal ↔ Casual | Button labels, microcopy, headings |
| Complexity | Simple ↔ Technical | Jargon, tooltip depth, glossary needs |
| Character | Serious ↔ Playful | Error tone, illustrations, motion style |
| Emotion | Reserved ↔ Expressive | CTA urgency, celebration, feedback |
| Authority | Advisory ↔ Commanding | Navigation labels, confirmations, warnings |

Choose 3-5 voice traits and write do/don't pairs:

| Trait | Do | Don't |
|---|---|---|
| Confident | "Deploy with rollback ready" | "Obviously the best deployment tool" |
| Friendly | "Let's fix this field" | "Oopsie, you broke it" |
| Precise | "Export CSV" | "Get stuff" |

## Persuasive Copy Patterns

### PAS — Problem, Agitate, Solution

Use for problem sections and high-friction landing pages.

`[Pain]? Every [timeframe], [consequence]. [Solution] fixes this by [mechanism].`

### AIDA — Attention, Interest, Desire, Action

Use for CTAs and closing sections.

`[Bold claim]. [Specific benefit]. [Proof]. [Action].`

### FAB — Feature, Advantage, Benefit

Use for feature cards.

`[Feature] lets you [advantage], so you can [benefit].`

### Before / After / Bridge

Use for transformations and case studies.

`Before: [pain]. After: [desired state]. Bridge: [product/mechanism].`

### Cost of Inaction

Use carefully for urgency.

`Without [change], [audience] loses [cost/risk] every [period].`

## CTA Design

- One primary CTA per surface section.
- Match CTA to readiness: Learn → Compare → Try → Buy → Continue → Confirm.
- CTA text should answer “what happens next?”
- Place CTAs after sufficient value/proof for the risk level.
- Destructive CTAs must name the consequence.

## Brand Consistency Review

- Visual style matches stated voice traits.
- Color and typography follow brand or declared local direction.
- Terminology is consistent across headings, buttons, forms, and errors.
- Differentiator is visible, not buried in prose.
- Competitors could not say the exact same headline unchanged.
- Claims have nearby proof.
- CTAs are consistent in language and hierarchy.
